# aptly-ui

Лёгкий веб-интерфейс для управления Aptly (создание локальных репозиториев, копирование/удаление пакетов, упрощённая работа с публикациями).

Описание
--------
Приложение написано на Flask и выступает как UI + прокси к Aptly API. Интерфейс отображает
операции создания репозиториев, копирования пакетов, удаления и даёт пользователю готовые curl-
команды для ручного исполнения. Поддерживает интернационализацию (i18n) и ролевую модель доступа.

Возможности
-----------
- Создание и удаление репозиториев
- Копирование пакетов между репозиториями
- Управление публикациями
- Авторизация пользователей
- Многоязычный интерфейс (en/ru)
- Логирование действий пользователей
- Docker-ready развертывание

Структура проекта
------------------
- `app.py` — основной файл приложения.
- `routes/` — модули с маршрутами Flask (auth, api_requests, create_repo и др.).
- `static/` — статические файлы (CSS, JS, favicon).
- `templates/` — HTML-шаблоны для страниц.
- `i18n/` — файлы локализации (en.json, ru.json).
- `logs/` — директория для логов приложения.
- `users.json` — файл с данными пользователей.
- `aptly-ui.conf` — конфигурационный файл.
- `requirements.txt` — зависимости Python.
- `Dockerfile` — для сборки Docker-образа.
- `Makefile` — скрипты для сборки и запуска.

Ключевые файлы
---------------
- `app.py` — основной Flask-приложение (backend). Прокси и бизнес‑логика.
- `static/aptly-ui.js` — frontend-логика (генерация команд, автокомплиты, UI handlers).
- `templates/` — Jinja2-шаблоны для страниц.
- `aptly-ui.conf` — конфигурация приложения (API_URL и пр.).
- `requirements.txt` — список зависимостей.


Конфигурация
------------
Файл конфигурации — `aptly-ui.conf` (в корне проекта). Пример (значения по умолчанию закомментированы):

```
# API_URL = http://localhost:8080/api
# PUBLISH_ARCH = amd64
# PUBLISH_ORIGIN = MyOrg
# PUBLISH_LABEL = MyLabel
# ALLOWED_CODENAME = bionic,jammy
# LOG_LEVEL = INFO
# LOG_MAX_BYTES = 10485760
# LOG_BACKUP_COUNT = 5
```

Настройки:
- `API_URL` — базовый URL Aptly API (например `http://localhost:8080/api`)
- `PUBLISH_ARCH` — архитектура для публикации (по умолчанию amd64)
- `PUBLISH_ORIGIN` — указывает источник публикации
- `PUBLISH_LABEL` — метка публикации
- `ALLOWED_CODENAME` — список допустимых кодовых имён через запятую (например: bionic,jammy)
- `LOG_LEVEL` — уровень логирования (INFO, DEBUG, WARNING, ERROR)
- `LOG_MAX_BYTES` — максимальный размер лог-файла до ротации
- `LOG_BACKUP_COUNT` — количество файлов ротации логов

Aptly publish encoding
----------------------
При работе с Aptly публикациями (эндпоинты `/api/publish`) префиксы и дистрибуции кодируются
по официальному правилу:
- префикс: все `_` заменяются на `__`, а `/` заменяются на `_` (пример: `foo/bar` -> `foo_bar`);
- distribution: применяется URL-энкодинг (процентное кодирование), в frontend используется `encodeURIComponent`, в backend — `urllib.parse.quote(..., safe='')`.


Запуск
------
Все команды для запуска и сборки есть в `Makefile` — предпочтительный способ управления сервисом.

**Локально (разработка):**

```bash
# создать venv и установить зависимости
make install

# запустить Flask приложение (использует venv)
make run
```

Приложение запустится на `http://127.0.0.1:5000`

**Через Docker / Gunicorn:**

```bash
# собрать Docker-образ
make build

# запустить контейнер (смонтирует ./aptly-ui.conf в /app/aptly-ui.conf внутри контейнера)
make start

# просмотр логов контейнера
make logs

# остановить контейнер
make stop
```

Альтернативные цели: `make up` (запуск), `make down` (остановка).

Управление пользователями
-------------------------
- Только пользователь с `id == 1` (admin) может менять настройки и управлять пользователями
- Все пользователи хранятся в файле `users.json` в корне проекта в формате JSON
- Пароли хранятся в хешированном виде (bcrypt)
- Поддерживается сессионная аутентификация через Flask-Login

Логи
----
В папке `logs/` хранятся следующие файлы:
- `app.log` — технические события и ошибки приложения
- `auth.log` — события аутентификации и управления пользователями
- `commands.log` — действия пользователей (создание/копирование/удаление)

Параметры ротации настраиваются в `aptly-ui.conf`:
- `LOG_MAX_BYTES` — максимальный размер файла до ротации
- `LOG_BACKUP_COUNT` — количество сохраняемых архивных файлов
- `LOG_LEVEL` — уровень детализации логов

Разработка и отладка
--------------------
- Код написан с прицелом на простоту. Комментарии и документация на русском внутри `app.py` и `static/aptly-ui.js`.
- Если вы правите код фронтенда — после изменений надо перезагрузить страницу в браузере (иногда полезно жесткое обновление).

Makefile
--------
В репозитории есть `Makefile` с удобными целями для разработки и деплоя. Доступные цели:

**Разработка:**
- `install`, `venv` — создать виртуальное окружение и установить зависимости
- `run` — запустить Flask приложение локально (http://127.0.0.1:5000)
- `clean` — очистить временные файлы, __pycache__ и виртуальное окружение

**Docker:**
- `build` — собрать Docker-образ (aptly-ui:latest)
- `start`, `up` — запустить контейнер Docker
- `stop`, `down` — остановить и удалить контейнер
- `logs` — просмотреть логи контейнера
- `cli` — открыть интерактивный shell в контейнере

**Утилиты:**
- `help` — вывести справку по доступным целям
- `print-VARIABLE` — вывести значение переменной (для отладки)
